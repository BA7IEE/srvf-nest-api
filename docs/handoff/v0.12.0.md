### 4.3 production storage_settings fail-fast(沿 PR #102)

`StorageSettingsService.onApplicationBootstrap()` hook,**严格 `env === 'production'` 守卫**(smoke 必须跳过),**5 项校验缺一启动失败**:

1. **settings 存在**(运维真实 PATCH 创建过 row)
2. **enabled === true**
3. **providerType === 'COS'**(production 拒绝 LOCAL;沿 F2)
4. **bucket / region 非空**
5. **credentialStatus === CONFIGURED**(凭证已录入 + 解密成功)

任一失败 throw Error → Pod CrashLoop → 运维 / on-call 立即收到告警。错误消息含修复指引(指向 [`docs/ops/cos-production-rollout-checklist.md §7 / §8`](../ops/cos-production-rollout-checklist.md));**永不**包含凭证 secret 明文 / 密文(沿 §6.6 信息泄漏防御)。

成功路径 logger.log 仅含状态字段(`providerType` / `bucket` / `region` / `credentialStatus`)。

### 4.4 AppEnv 'smoke' + isProductionLike helper(沿 PR #102)

- **`VALID_APP_ENVS` +1 'smoke'**(`src/config/app.config.ts`):CI Docker smoke job 专用 AppEnv;**不得用于真实部署**
- **`isProductionLike(env)` helper**:`env === 'production' || env === 'smoke'`
- 设计哲学:smoke 几乎完全沿 production 行为,**唯一例外** = storage_settings fail-fast

### 4.5 6 处 production-like 守护改造(沿 PR #102)

| # | 位置 | 改造 |
|---|---|---|
| 1 | `LOG_LEVEL` 默认 = `info` | smoke / production = info |
| 2 | `STORAGE_ENCRYPTION_KEY` 必填 | smoke / production 必填 |
| 3 | CORS 严格(禁空 / 禁 `*`)| smoke / production 严格 |
| 4 | swaggerEnabled 默认禁 | smoke / production 默认禁 |
| 5 | AllExceptionsFilter 隐藏 message | smoke / production 隐藏 |
| 6 | logger isProd(JSON / 不开 pino-pretty)| smoke / production 一致 |

**关键铁律**:`isProductionLike` **不可用于 storage_settings fail-fast**(必须 `env === 'production'` 严格守卫;否则 docker-smoke CI 会在 `APP_ENV=smoke` 下被 fail-fast 误挡)。

## 9. 已知 quirk:`gh pr merge` worktree checkout main 冲突预案

本仓库在 `.claude/worktrees/` 下创建临时 worktree 时,`gh pr merge --squash --delete-branch` 会撞:

```
failed to run git: fatal: 'main' is already used by worktree at '/Users/.../srvf-nest-api'
```

**根因**:主仓 worktree 占用 `main` 分支,gh 试图本地 checkout main 失败。

**预案**(沿 PR #54-#103 累计 ≥ 28 次验证成功):

```bash
# 1. 仍执行 gh pr merge(GitHub 端会完成 squash merge)
gh pr merge <PR#> --squash --delete-branch
# → 报 worktree 错误,但 GitHub 端已 MERGED

# 2. 验证 GitHub 端 MERGED 状态
gh pr view <PR#> --json state,mergedAt,mergeCommit

# 3. 检查远端分支是否已删
git ls-remote --heads origin <branch-name>

# 4. 手动删除远端分支
git push origin --delete <branch-name>

# 5. 主仓库 ff main 到最新
git -C /Users/dengwang/Documents/coding/srvf-nest-api pull --ff-only origin main

# 6. worktree 同步到 main(开下一 PR 前)
git fetch origin main && git reset --hard origin/main
```

详细步骤见 v0.8.1 handoff §11(预案不变)。
