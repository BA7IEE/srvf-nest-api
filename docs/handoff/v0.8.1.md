## 11. 工作流速查(worktree 环境特殊处理)

本仓库在 `.claude/worktrees/` 下创建临时 worktree 时,**`gh pr merge --squash --delete-branch` 会撞** `fatal: 'main' is already used by worktree at '...'` 错误(主仓 worktree 占用 `main` 分支)。

**预案**(已在 PR #44-#48 5 次循环验证):

```bash
# 1. 触发 squash merge(预期撞错;但 GitHub 端已成功 squash merge)
gh pr merge <PR_NUM> --squash --delete-branch
# 报错:fatal: 'main' is already used by worktree at '/Users/dengwang/Documents/coding/srvf-nest-api'

# 2. 确认 GitHub 端 MERGED
gh pr view <PR_NUM> --json state,mergeCommit

# 3. 手动删除远端分支(GitHub 端 squash merge 时未删,需补)
git push origin --delete <branch-name>

# 4. 主仓 fast-forward
git -C /Users/dengwang/Documents/coding/srvf-nest-api pull --ff-only origin main
```
