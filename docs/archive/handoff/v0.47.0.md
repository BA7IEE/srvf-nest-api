## 3. 部署与联调提示

- 部署前审查并按生产 SOP 应用第 50 migration `add_insurance_expiry_notification_markers`;两列均 nullable additive,无回填 SQL、无不可逆操作。**不要**在未审查环境执行 reset。
- 到期 job 每日 09:00 `Asia/Shanghai` 运行;上线后首跑会处理已进入窗口/已到期的存量。站内通知无需额外 provider;微信 `expiry-reminder` 模板配置为空时仅跳过微信 best-effort,不阻断站内通知或证书到期状态迁移。
- 管理后台新增考勤撤回操作时必须提交非空 `reason`;仅 approved 单可操作,调用者需 `attendance.reopen.sheet` scoped 权限。前端撤回后应按 pending 流重新编辑/一审/终审,不要假设原 records 被删除。
- 小程序/H5 将证书大类 `first_aid` 展示为「急救资质」;招新轮次人数输入留空或清空均表示不限。内部 `redCross` code 与既有历史数据不迁移。
- 不回改本 handoff;发布后事实由 `docs/current-state.md §1/§2/§4` 承接。
