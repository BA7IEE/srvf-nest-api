## 2. 本版交付

- Auth 密码、短信、微信签发与 refresh rotation/replay、logout、改密/重置、身份变更、禁用/软删和队员离队统一以 PostgreSQL User 行锁线性化，并在锁后复验 ACTIVE、`deletedAt`、`passwordHash`、phone、openid；既有 opaque refresh、family revoke、90 天 absolute、错误码、JWT payload 与 access-token 策略不变。
- JWT access/refresh TTL 在启动期解析并做范围校验，运行时响应提供数值化 `expiresInSeconds` / `refreshExpiresInMs`；无单位、非法或越界配置 fail-fast。
- production Storage 离线 bootstrap 使用窄配置边界；disabled 设置在 API/worker 重启后仍保持普通 Effect fail-closed，production 不回退 Local，Provider location 在既有人工 relocation 流之外冻结。
- 所有 366 个 OpenAPI operation 的成功状态与 Nest 实际状态全等：38 个 201、3 个 204、2 个 CSV 200，并明确 4 个 settings GET 的 nullable data；Unit / Contract / E2E 与 Docker image build 均全绿。
- 自动日志不再记录 HTTP query string/search value，同时保留 method、pathname、status、response time、request ID 与 authenticated user ID。
- `fast-uri` override 从 `^3.1.2` 提升到 `^3.1.4`，清除 `GHSA-v2hh-gcrm-f6hx` High；未升级 COS SDK、conf 或 ajv。Release audit workflow 移除无用 pnpm cache，Docker Smoke 增加 `workflow_dispatch`。

## 4. 上线硬门(本版未完成)

- **真实数据与制品**:目标 production DB 的 migration/seed/bootstrap、最终 registry image tag + digest、备份/回滚与生产开关必须另行现场复核；release/tag 不等于部署。

- **生产拓扑**:真实 ingress/client IP、精确 trusted proxy CIDR、edge 覆盖、backend ACL、所有 API/worker 实例配置一致、旧实例=0、混档禁止与 worker 切换票。
- **外部通道**:微信小程序、腾讯云 OCR/SMS 的真实凭证、计费调用、失败/恢复与审计验收按各自 `docs/ops/` SOP 执行。
- **公开招新**:首发默认关闭；除非另行完成 durable ledger 或可靠 COS prefix 与数据库 key reconciliation，ingress 必须技术拒绝 `/api/open/v1/recruitment/**` 并提供外网不可达证据，不能只隐藏前端入口。

上述任一缺失时，状态只能是“代码/release 自动化 ready，production GO blocked”，不得写成已上线。
