## 3. 部署与联调提示

- **无需数据库操作**:本 release 无 schema / migration / seed / 数据回填,禁止为旧轮做人工批量订正。
- 小程序候选部门选择器须限制最多 2 项;发起/修改传 3 项及以上会收到 HTTP 400 / code 40000。读取旧申请时仍可能看到 >2 个历史候选,前端应照常展示,不要自行裁剪。
- 管理后台开轮/编辑轮表单须将 `maxTargetOrgs` 域改为 1..2,null 表示默认 2。旧轮详情可能仍回显数据库原始值 >2,但 App 实际有效上限固定按 2。
- 贡献值展示直接使用后端返回值,不要在前端裸 SUM。v0.48.0 起历史考勤也按每日上限 3 实时重算,因此 App 入队进度与 Admin 队员 360 数字可能变大。
- 不回改本 handoff;发布后事实由 `docs/current-state.md §1/§2/§4` 承接。
