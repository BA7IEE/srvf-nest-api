## 3. 部署与前端联调

- **部署零动作**:无 migration、无 seed 变更、无新 env、无 cron 变更;常规 `prisma migrate deploy` 确认 no pending 即可。
- **admin-web(全部 additive,按需接入)**:活动详情页可接 reconciliation(缺席名单/临时参加)与 participation-summary(到场率/合计/直方图);队员 360 可加 participation-summary tab(与既有 contribution-summary 并存,后者原样);审批工作台可接 bulk-approve/bulk-reject(注意:响应是 succeeded/failed 清单,失败条目逐条给 BizCode,前端应逐条展示而非整包报错);仪表盘可加 participation-overview 月度报表。详见 [`docs/handoff/admin-web.md`](../../handoff/admin-web.md)。
- **miniapp**:`GET app/v1/my/participation-summary` 返回本人累计时长/次数/记录数/封顶贡献四字段,可直接做「我的志愿档案」入口。详见 [`docs/handoff/miniapp.md`](../../handoff/miniapp.md)。
- 不回改本 handoff;发布后事实由 `docs/current-state.md §1/§2` 与 `docs/handoff/openapi.json` 承接。
