## 1. TL;DR

- **v0.29.0 = 招新实名环节 OCR 改造**(单一 goal;二要素**真实性核验** → 腾讯云 **OCR 多证件识别 + 自洽匹配**;**明确放弃联网真实性核验**)。
- **通道层语义换血**(#427):`realname/` Provider 契约 `verify(name,idCard)→{matched}` 改 `recognize(documentTypeCode,image)→{结构化字段 + 防伪 warnings + 清晰度}`;三 action 按证件类型分流(`RecognizeValidIDCardOCR` 身份证〔自带防伪〕/ `MLIDPassportOCR` 护照〔仅机读〕/ `MainlandPermitOCR` 回乡证〔仅来往内地〕)走 `ocr.tencentcloudapi.com`(service `ocr` / version `2018-11-19`),**复用 TC3-HMAC-SHA256 签名**(`buildSignedHeaders` 参数化 action,零新依赖);**全仓删除 `IdCardVerification` 调用路径**(grep 自证仅注释);真通道休眠(DevStub OCR 桩 + `.spec` mock fetch 锁三 action)。
- **报名流程重构 + 状态机净化**(#427):**OCR 前置 + 单事务建终态**;`mainland_id` 匹配 + 防伪无告警 + 清晰 → 自动 `verified` 发临时编号,不匹配 / 告警 / 不清晰 / OCR 失败 → `manual_review`(**不再 `rejected`**,「对不上转人工不误杀」);护照 / 回乡证 → `manual_review`(识别端 OCR 回填 + 人工最终,提交端不重识别);台胞证 / 外国人永居 / 其余 → `manual_review` 不 OCR。**退役 `pending_verification` 在途态 + FM-A 卡死类**(OCR 移到唯一事务之前,失败整体回滚无残留)。姓名 NFC 完全一致。
- **新增公开识别端点**(#427):`POST /api/open/v1/recruitment/applications/recognize`(`@Public` 第 9 throttler;无状态扫证件回填供确认;非 OCR 类型 `ocrSupported:false`,不清晰 `clarityOk:false`;27030/27031 仅识别端浮现)。`verifyOutcome` 加细分 String 值(`forgery_warning` / `ocr_unclear` / `ocr_error` / `category_mismatch`)。
- **六分叉拍板冻结**(评审稿 [`recruitment-realname-ocr-review.md`](../reviews/recruitment-realname-ocr-review.md) §0.5,2026-06-22「按推荐」+ 4 把关):①无 HMAC token + 提交端重识别 ②仅 mainland_id 重识别 ③提交端 OCR 永不硬报错转人工 ④消灭 pending_verification/FM-A ⑤姓名 NFC 完全一致 ⑥verifyOutcome 细分。
- 收口链:base v0.28.0(`e4bd6d5e` / tag)→ #427(`4958df43` T1+T2)+ #428(`6e65f8ef` T3 docs)→ bump #429(`4dcd9c48` = **tag 点**)→ §1 回填 + 本 handoff(post-release docs PR)。
- `v0.29.0` tag(annotated,指 `4dcd9c48`)+ GitHub Release(**Latest**)+ current-state §1 **均已完成**。
