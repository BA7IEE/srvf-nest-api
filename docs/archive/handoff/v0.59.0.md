## 3. 上线边界(Release 不等于部署)

- **未做生产部署、生产 migration、gate enable、旧 server 排空或 tag 前现场切流**。保险 v3 约束与 enforcement gate、notification outbox/G2 migrations 均尚未在生产生效；上线前必须确认旧 API/worker=0、排空旧事务与不兼容 active intents，并禁止混合 producer/worker 或混档 gate。
- 可信代理代码边界只接受 `none` 或精确 canonical CIDR，production/smoke 缺失拒启；真实生产 CIDR、edge 覆盖与 backend ACL 仍须现场验证，反代部署不得用 `none`。
- Attachment 当前仍是 Phase 1：durable ledger 与 Content publish/confirm 根锁已接线，但 `Attachment.key` FK、repo-wide raw-key 引用闭环及 Provider 原子性不在本版完成声明内。
- published Content 全字段不可变、完整 dictionary in-use 防护、participation 类通知 producer 余面仍属后续；详见 `docs/ai-harness/NEXT_TASKS.md` 与 release 后 `docs/current-state.md`。
