## 2. 本版交付

- `storage-settings:bootstrap` 支持全新 production 空库在应用启动前以私密 JSON、目标数据库二次确认、dry-run、空表拒覆盖、事务内写后读取与凭证解密完成一次性初始化；零 schema/migration/seed/API 变更。
- production boot/PATCH 强制 `COS + enabled=true + bucket/region + 可解密凭证`；`enabled=false` 是显式 kill switch，普通 API/worker effect 全部 fail-closed，只有人工复核的 `manual_relocate` 证据路径可显式绕过。
- COS SOP 以当前 DTO/OpenAPI 重写为 migration → seed → offline bootstrap → production boot → login → GET → upload-url → PUT → confirm/HEAD → download → DELETE，并用真实 ValidationPipe 与 Nest test app fixture 锁定字段、状态、权限码和去敏响应。
- `STORAGE_ENCRYPTION_KEY` / `SMS_ENCRYPTION_KEY` / `WECHAT_ENCRYPTION_KEY` / `REALNAME_ENCRYPTION_KEY` 首发值互异并冻结；当前无 key version、双 key 或离线重加密工具，禁止把 reset-credentials 当作 encryption-key rotation。
- `POST /api/auth/v1/logout` 的 OpenAPI、handoff 与 active auth reference 对齐已落地的 family 撤销、幂等 `data:null` 与“实际撤销数 > 0 才审计”；运行时行为未在该校准 PR 中改变。
- `.env.example`、CODEMAP、部署 tag 示例与保险 PR1–PR4 状态完成 true-up；新增 `pnpm audit:production` 和 tag/manual workflow，以 high/critical 阻断 production 依赖，moderate/low 必须逐条登记。

## 4. 上线硬门(本版未完成)

- **真实 COS**:私有 ACL、CAM 最小权限与跨 bucket 拒绝、CORS、versioning、lifecycle、SSE-COS，以及 upload-url → PUT → confirm 内部 HEAD → signed download → DELETE 的最终镜像全链证据。
- **生产拓扑**:真实 ingress/client IP、精确 trusted proxy CIDR、edge 覆盖、backend ACL、所有 API/worker 实例环境一致性、旧实例=0 与混档禁止。
- **生产数据/开关**:migration/seed/bootstrap 的真实目标复核、保险 gate enable 与旧 server/事务 drain、通知/outbox 与 storage worker 切换票。
- **制品**:最终 registry image tag + digest；本地 Docker build 的 image ID 不能替代 registry digest。
- **外部通道**:微信小程序、腾讯云 OCR/SMS 的真实凭证与计费调用验收按各自 `docs/ops/` SOP 执行。

上述任一缺失时，状态只能是“repository/release ready，production GO blocked”，不得写成已上线。
