# Codex 交叉评审 SOP(Harness 2.0)

> 依据:公理 A5 —— 同一模型自写自查,错误相关会一起漏;写与查必须跨模型。协议条文在 [`process.md §8.3`](../process.md);本页是操作 SOP。仓库 `AGENTS.md` 恰是 Codex 原生自动读取的规则入口,无需额外配置。

## 1. 何时评审

- **必须**:红区 / D 档 PR(schema · auth · 权限 · harness 文档重写 · workflows)。
- **尽量**:B/C 档实质 PR 合并前过一遍。
- **可免**:A 档 docs-only。

## 2. 怎么投放(维护者动作,约 1 分钟)

把 PR 链接交给 Codex(CLI / IDE / 云端任一形态),附模板:

> 请评审 <PR 链接>。要求:
> ① 对照 PR body「写集声明」核对 diff 是否越集;
> ② 找正确性 / 安全 / 并发问题,给"文件:行号"证据;
> ③ 对照 AGENTS.md §1 铁律速查与 §2 决策锁,标注任何违反;
> ④ 结论分级 BLOCKER / SHOULD-FIX / NIT;没有问题就明说,不要凑数。
> findings 以评论发在 PR 上(或输出给我粘贴到 PR)。

## 3. findings 处置(总控)

- 落点统一 = **PR 评论**;总控逐条消化:接受 → 修并回帖;不接受 → 评论给理由。
- **分歧不内部调和**:两模型结论相反 → 原样升级进人话简报由维护者裁(分歧点 = 维护者注意力的优先级信号)。
- findings 与处置留在 PR 记录,不进仓库正文。

## 4. 边界

- Codex 评审是只读的,不授予直接改仓库。
- Codex 承接执行 lane 时(限 A/B 档,goal 立项),对称规则生效:Claude 评审其 PR。
- 本页更新走 A 档 PR;与 process §8 冲突时本页让步。
