## 0. 文档定位

### 0.1 这份文件是什么

- 18 道**选择题 / 半选择题**,**控制在 20 题以内**
- 每题都是"会直接决定数据模型形态"的关键问题
- 设计为"被访谈人 20 分钟以内能答完"的负担级
- 答完即可让 V2 候选模型草案(`data-model-draft.md`)的关键节点从 [待调研] 变成 [当前倾向] 或 [已确认]

### 0.2 这份文件不是什么

- **不是**完整调研版 — 完整版见 `research-questions.md`(82 题,9 节,涵盖所有 [待调研] 项)
- **不是**业务需求文档 — 不询问"系统应该有什么功能",只问"组织结构与数据现状的事实"
- **不是**最终决策文档 — 本文产出的答案需要回填到 `research.md §4` 与 `data-model-draft.md §3` 才算完成

### 0.3 严守的边界

- **不**让被访谈人看技术术语(Prisma / schema / API / module / SQL 等一律不出现在题干)
- **不**写真实部门名 / 真实等级取值 / 真实活动名 / 真实证书名 — 题干用语义类别(救援出动 / 训练 / 会议)
- **不**追问业务流程细节,只问"是 / 否 / 数量 / 类别"等可勾选答案
- **不**在仓库中记录被访谈人的真实回答 — 答案以另存为方式留存(如运营私有空间),只把"形态级结论"回填到 `research.md`

## 1. 使用方式

### 1.1 角色分工

| 角色 | 职责 |
|---|---|
| **被访谈人**(救援队负责人) | 按题目选择最贴近的选项;不需要看 📌 标签;不知道就选"不确定";可以让访谈记录人事后补充 |
| **访谈记录人**(项目协作者) | 用**业务语言**逐题提问;勾选答案;答完后按 📌 标签把答案翻译成 `research.md §4` 与 `data-model-draft.md §3` 可用的形态级结论 |
| **AI 助手**(后续回填) | 收到记录人整理后的"结论摘要",做 research.md / draft.md 的回填(走 docs 同步任务) |

### 1.2 时长建议

- **总时长**:20 分钟以内
- **每题**:平均 1 分钟
- **如某题被访谈人需要查资料才能答**:让访谈记录人标"待补"先跳过,继续后续题目

### 1.3 作答标记规则

每题作答时勾选选项即可,不需要写长答案。

仅 **Q8 / Q9 / Q10**(敏感信息三题)在选 ① / ② 时**额外要求**简短回答三问(业务用途 / 查看角色 / 保存期限),每问一句话即可,合规未明确则选 ④。

### 1.4 答完后的处理

访谈记录人在每题下方记录答案,然后按 📌 标签:

1. 把答案翻译为 `research.md §4.x` 对应小节的"调研结论"段
2. 把对应 `data-model-draft.md §3.x` / §4.x 的 [待调研] 替换为 [当前倾向] 或 [已确认]
3. 真实运营信息(具体名字 / 取值列表)**不进**仓库,通过运营后台 / 私有 seed 承载(对应 `research.md §5.1` / §7-R13)
